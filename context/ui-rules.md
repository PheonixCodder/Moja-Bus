# UI Rules

## Global Layout Rules
- Passenger web should feel direct, lightweight, and search-first.
- Passenger mobile should prioritize search, seat selection, checkout, and offline ticket access.
- Operator/admin web should feel dense, utilitarian, and work-focused.
- Use responsive layouts that work on low-end phones and large desktop displays.

## Interaction and States
- Every interactive control needs hover, focus, active, disabled, and loading states.
- Seat maps and manifest views must keep fixed geometry.
- Offline and sync states must be visible and explicit.
- Payment and booking actions must always show confirmation or failure feedback.
- Operator/admin actions should make role and permission boundaries obvious.

## Animations and Transitions
- Keep motion subtle and purposeful.
- Use small fades, slides, and loading skeletons only when they help comprehension.
- Do not use decorative motion as the main design feature.
